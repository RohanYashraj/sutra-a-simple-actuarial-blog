# Sutra — Actuarial Intelligence, Simplified

Sutra is an AI-powered blog and newsletter platform that delivers daily insights at the intersection of **insurance, actuarial science, AI, finance, and technology**. Built with Next.js, powered by Google Gemini, and delivered via Resend.

**Live at** [sutra.rohanyashraj.com](https://sutra.rohanyashraj.com)

---

## Content Streams

Sutra publishes six themed newsletter streams, each generated by a shared content engine and delivered on a weekly schedule:

| Stream | Route | Focus | Day (UTC) |
| :--- | :--- | :--- | :--- |
| **Sutra Trivia** | `/api/trivia` | Weekly actuarial/insurance + AI insight | Wednesday |
| **Market Pulse** | `/api/market-pulse` | Economic & market trends with insurance implications | Saturday |
| **Code Sutra** | `/api/code-sutra` | AI-augmented coding for actuaries & insurance engineers | Tuesday |
| **GenAI Frontiers** | `/api/genai-frontiers` | Frontier AI/agentic AI news with actuarial applications | Friday |
| **Actuarial Simplified** | `/api/actuarial-simplified` | One concept tied to a current event, explained simply | Monday & Thursday |
| **Sutra Digest** | `/api/digest` | Periodic top article summaries | Monday |

Every edition is archived at `/archive/[type]/[slug]` so subscribers can revisit past issues.

## Architecture

```
┌─────────────────┐     ┌──────────────────┐     ┌───────────────┐
│  cron-job.org    │────▶│  /api/cron        │────▶│  Stream Route  │
│  (every 15 min)  │     │  (orchestrator)   │     │  e.g. /api/    │
└─────────────────┘     └──────────────────┘     │  trivia        │
                                                  └───────┬───────┘
                                                          │
                                    ┌─────────────────────┼─────────────────────┐
                                    ▼                     ▼                     ▼
                            ┌──────────────┐    ┌────────────────┐    ┌──────────────┐
                            │ Content Engine│    │  Email Template │    │   Convex DB   │
                            │ lib/gemini.ts │    │  lib/email.ts   │    │  broadcasts   │
                            │  (Gemini API) │    │  (Resend Send)  │    │  (archive)    │
                            └──────────────┘    └────────────────┘    └──────────────┘
                                                                              │
                                                                              ▼
                                                                    ┌──────────────────┐
                                                                    │ /archive/[type]/  │
                                                                    │ [slug] (Next.js)  │
                                                                    └──────────────────┘
```

### Key Modules

- **`lib/gemini.ts`** — Centralized content engine. Each stream has a dedicated prompt config; a shared `callGeminiJson<T>()` helper handles retries, JSON extraction, and validation. `generateStreamContent(streamId)` is the single entry point.
- **`lib/email.ts`** — Base HTML email template with a modern, minimal design. Mobile-responsive (full-width on small screens, card layout on desktop).
- **`lib/slug.ts`** — URL-safe slug generation via `sanitizeSlug()`.
- **`lib/indexnow.ts`** — Submits new archive URLs to search engines.
- **`convex/broadcasts.ts`** — Convex mutations/queries for persisting and retrieving broadcast editions.

## Getting Started

### Prerequisites

- Node.js 18+
- A [Convex](https://www.convex.dev/) project
- API keys for Gemini, Resend, and (optionally) PostHog

### Environment Variables

Create a `.env.local` file:

```env
GEMINI_API_KEY=           # Google Gemini API key
GEMINI_MODEL=             # Optional — defaults to gemini-3-flash-preview
RESEND_API_KEY=           # Resend API key
RESEND_AUDIENCE_ID=       # Resend audience list ID
CRON_SECRET=              # Secret to protect the /api/cron endpoint
CONVEX_URL=               # Your Convex deployment URL
NEXT_PUBLIC_CONVEX_URL=   # Same, for client-side usage
```

### Install & Run

```bash
pnpm install
pnpm dev
```

## Cron Orchestration

The `/api/cron` route acts as a time-based orchestrator. It is pinged externally (e.g. via [cron-job.org](https://cron-job.org)) every 15 minutes and checks the current UTC time/day to decide which stream to trigger.

### Security

The endpoint requires either:
- A `secret` query parameter matching `CRON_SECRET`, or
- An `Authorization: Bearer <CRON_SECRET>` header.

### Setting Up cron-job.org

1. Generate a secret: `openssl rand -base64 32`. Add it to Vercel as `CRON_SECRET`.
2. Create a job on cron-job.org:
   - **URL**: `https://sutra.rohanyashraj.com/api/cron?secret=YOUR_SECRET`
   - **Schedule**: `0,15,30,45 * * * *` (every 15 minutes)
   - **Method**: GET

### Broadcast Schedule (UTC 08:15 window unless noted)

| Stream | Day(s) | UTC Window |
| :--- | :--- | :--- |
| Actuarial Simplified | Mon, Thu | 08:10–08:20 |
| Code Sutra | Tue | 08:10–08:20 |
| Sutra Trivia | Wed | 08:10–08:20 |
| GenAI Frontiers | Fri | 08:10–08:20 |
| Market Pulse | Sat | 08:10–08:20 |
| Sutra Digest | Mon | 13:10–13:20 |

## Testing & Manual Triggers

### Force a Specific Stream

Use the `force` query parameter to bypass the schedule and trigger any stream immediately:

```bash
# Via curl
curl "https://sutra.rohanyashraj.com/api/cron?force=trivia&secret=YOUR_SECRET"

# Available force values:
#   trivia, digest, market-pulse, code-sutra, genai-frontiers, actuarial-simplified
```

### Hit a Stream Endpoint Directly (local dev)

```bash
curl http://localhost:3000/api/trivia
curl http://localhost:3000/api/market-pulse
curl http://localhost:3000/api/code-sutra
curl http://localhost:3000/api/genai-frontiers
curl http://localhost:3000/api/actuarial-simplified
curl http://localhost:3000/api/digest
```

### Manual Audience Broadcasts

**Custom update email:**
```bash
# Edit SUBJECT and CONTENT_HTML in the script, then:
npx tsx scripts/broadcast-update.ts
```

**New article broadcast:**
```bash
npx tsx scripts/broadcast-article.ts <article-slug>
```
